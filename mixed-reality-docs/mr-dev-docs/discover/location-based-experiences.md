# <a name="location-based-entertainment-with-windows-mixed-reality"></a>Развлечения на основе расположения с использованием Windows Mixed Reality

В последние несколько лет мы видели невероятный объем роста и инноваций в категории "развлечения на основе местоположения". Традиционные места проведения, такие как темы парки и кинотеатры, начали предлагать впечатляющие, многопользовательские возможности в качестве бесплатных решений для существующих установок. Новые операторы и места проведения используют уникальные возможности работы с несколькими игроками по привлекательной цене на языке. Все эти возможности помещаются на конверте для того, что возможно в смешанной реальности.

Этот документ представляет собой руководство по началу работы с Windows Mixed Reality в категории "развлечения на месте". Приведенные ниже инструкции также могут быть применимы к средствам, основанным на местоположении, за исключением таких развлечений, как обучение, проектирование продукта и другие варианты использования.

### <a name="setup"></a>Настройка

**Вопрос. как легко настроить установку и настроить портал смешанной реальности для ЛБЕ?**

>[!NOTE]
>Для этой функции требуется версия 2000.19061.1011.0 или более поздняя.  

Может оказаться, что требуется больше настроек портала смешанной реальности, чем обычно доступно в приложении для развертывания приложений в киосках или настраиваемых интерфейсах. Последнее обновление портала Mixed Reality в июле поддерживает несколько дополнительных параметров, которые можно задать с помощью файла конфигурации:  

Разрешить неудачные проверки системы — обычно процесс установки проверяет компьютер на совместимость с Windows Mixed Reality перед завершением установки. Обход проверок совместимости может вызвать проблемы при попытке запуска Windows Mixed Reality при наличии проблем совместимости.  

Пропуск приложения-компаньона устройства — DCA предоставляет действия по настройке гарнитуры, предоставленные производителем, и позволяет обновлять встроенное по гарнитуры.  

Пропустить настройку комнаты. вместо того, чтобы получать запрос на создание границы комнаты, можно перейти непосредственно на гарнитуру в режиме «в помещении/в помещении».  

Пропустить установку приложений из магазина — обычная установка устанавливает несколько приложений Магазина, включая 3D Viewer и надстройку средства просмотра граничных 360. Это приведет к пропуску установки этих приложений, но может отсутствовать функциональность устройства.  

Предварительный просмотр в полноэкранном режиме. на портале Mixed Reality по умолчанию будет отображаться предварительный просмотр гарнитуры в полноэкранном режиме на настольном компьютере во время использования гарнитуры.  

Скрыть новые для своей боковой панели — предотвращает расширение новой панели при запуске портала смешанной реальности.  

#### <a name="how-to-configure"></a>Порядок настройки:  

Чтобы задать любую из этих конфигураций, необходимо создать файл с именем _UserConfig.json_ в этом каталоге: _$env: локалаппдата\паккажес\ Microsoft.MixedReality.Portal_8wekyb3d8bbwe \локалстате \\_

Для большинства пользователей это будет выглядеть следующим образом: _C:\Users \<username> \Аппдата\локал\паккажес\ Microsoft.mixedreality.portal_8wekyb3d8bbwe \локалстате \\_

Файл JSON должен иметь следующее содержимое со значением "true", установленным для любого из перечисленных выше параметров, которые должны быть включены:  

```
{

  "shouldAllowFailedSystemChecks": true,

  "shouldSkipDcaApp": true,

  "shouldSkipRoomSetup": true,

  "shouldSkipStoreAppInstall": true,

  "shouldShowPreviewFullScreen": true,

  "shouldHideEngagementPane": true

}
```
 
**Вопрос. есть ли какие-либо рекомендации по настройке плайспаце?**

Ответ. Настройка плайспаце должна выполняться так же, как и при настройке потребителя. Процесс установки комнаты также позволит определить границы комнаты. Дополнительные сведения о настройке границ комнаты можно прочитать [здесь](https://docs.microsoft.com//windows/mixed-reality/enthusiast-guide/set-up-windows-mixed-reality#set-up-your-room-boundary).

Как обсуждалось в приведенном выше документе, максимально разумная однокоординатная плайспаце обходится вокруг 5mx5m. Если вы хотите увеличить область, можно использовать функцию пространственных привязок в стеке API Windows Holographic. Использование этого API потребует пользовательского проектирования в создаваемых вами возможностях.  

Дополнительные сведения о том, как оптимизировать содержимое для различных размеров пространства, можно прочитать [здесь](https://docs.microsoft.com//windows/mixed-reality/coordinate-systems).
 

**Вопрос. пространство на диске слишком велико, и при попытке настроить работа с границами возникают ошибки. Что делать, чтобы настроить работу с большим объемом свободного роуминга?**

О. Чтобы настроить больший объем пространства, чем ~ 18x18ft, нельзя использовать границы, предоставляемые системой.  Граничные системы полагаются на одну фиксированную координату "Anchor", которая может стать нестабильной, если пользователь находится слишком далеко от привязки к центральному этапу. 

Можно настроить режим «под», который не будет отображать границу или настроить границы или плайспаце этапа.  Необходимо настроить несколько пространственных привязок в приложении, чтобы они ссылались на физические области границ. 

Разработчик приложения несет ответственность за отображение необходимых мер защиты, чтобы пользователи не могли конфликтовать с физической окружающей обработкой.  Это могут быть цифровые стены в рамках опыта или настраиваемый визуальный элемент "граница игры". 

Руководство по настройке границы комнаты для ВМР можно найти [здесь](https://docs.microsoft.com//windows/mixed-reality/enthusiast-guide/set-up-windows-mixed-reality#set-up-your-room-boundary).

**Вопрос. где находится источник плайспаце?**

Ответ. источник плайспаце определяется процессом настройки комнаты, точнее, в том ХМД, что при нажатии центральной кнопки во время установки. 

### <a name="multi-player-setup"></a>УСТАНОВКА С НЕСКОЛЬКИМИ ИГРОКАМИ

**Вопрос. я развертываю возможности работы с несколькими игроками в моем месте. Поддерживается ли в Windows Mixed Reality?**

Ответ. Если вы отказались от сборки Windows 20H1 или более поздней версии с помощью нашей программы предварительной оценки, то можете получить доступ к новому интерфейсу для совместного использования карт. Эта новая функциональность доступна через интерфейс [диспетчера карт](../develop/platform-capabilities-and-apis/using-the-windows-device-portal.md#map-manager) на портале устройств Windows. Чтобы использовать это средство, выполните следующие действия:
* Убедитесь, что вы выбрали 20H1 или более позднюю версию после сентября 2019. Это означает использование нашей программы предварительной оценки
* Включение портала устройств Windows (формате WDP) с помощью этих [инструкций](https://docs.microsoft.com/windows/uwp/debug-test-perf/device-portal-desktop)
* Подключите ХМД Windows Mixed Reality, чтобы загрузить существующую карту из или импортировать новую карту
* Перейдите к формате WDP в браузере по своему усмотрению, используя URL-адрес, указанный на экране параметры.
    * После этого перейдите к разделу "смешанная реальность" и выберите "Диспетчер карт".
    * Теперь можно использовать кнопку "Скачать", чтобы экспортировать существующую карту с компьютера.
    * С помощью кнопки "отправить файл отображения" можно импортировать карту из предыдущего экспорта (возможно, на другом компьютере).
    * Вы можете использовать "Импорт", чтобы система могла использовать эту карту для этого ХМД на этом компьютере.

> [!NOTE] 
> Раньше было возможно использовать средство упаковщика пространственных данных, однако это средство изначально было выпущено как неподдерживаемое и теперь официально устарело и больше не работает в 20H1. Вместо этого используйте диспетчер папки Inbox [Manager](../develop/platform-capabilities-and-apis/using-the-windows-device-portal.md#map-manager) , как описано выше. 

### <a name="tracking"></a>ОТСЛЕЖИВАЮЩ

Вопрос. как работает технология отслеживания на гарнитурах Windows Mixed Reality?  

Смешанная реальность использует ту же технологию отслеживания, что и HoloLens. Чтобы узнать больше о внутренней системе отслеживания, ознакомьтесь с документацией [здесь](https://docs.microsoft.com//windows/mixed-reality/enthusiast-guide/tracking-system).

Описание того, как работает система пространственного сопоставления более высокого уровня, можно прочитать [здесь](../design/spatial-mapping.md).

**Вопрос. Существуют ли рекомендации по получению надежного тома отслеживания?**

Чтобы лучше настроить среду для отслеживания успеха, можно ознакомиться с рекомендациями, приведенными в этой [записи](../environment-considerations-for-hololens.md).

**Вопрос. Существуют ли какие-либо конкретные особенности, связанные с отслеживанием дисковых пространств или оптимизацией, которые следует учитывать?**

Ответ. следующие рекомендации помогут получить более надежный том отслеживания:  

Предоставление различных функций в комнате, которые перекрываются в нескольких позициях, поможет системе отслеживания получить твердую блокировку. Представьте себе случайные цвета и штриховки, а не сплошные линии стиля контура. 

Сократите динамический диапазон освещения в своей области, где это возможно. Камеры отслеживания в нашей смешанной реальности ХМДС не HDR и имеют АГК (Автоувеличение) и AEC (автовыдержка), которые будут работать с различными освещения. В зависимости от распределения поверхности, шаблонов и контрастов, АГК или AEC могут задержать вас в большой черновой или черной комнате, которая может размыть функции, которые могут находиться в противоположном «цвете». Если вы пытаетесь получить внутреннюю картинку перед внешним окном с ярко-летним периодом, и вы настроите экспозицию так, чтобы вы могли видеть подробные сведения, то все во внутренней части будут подходящими и черними. Или если задано значение для внутри, то все, что находится за пределами, теперь является недоступным и все белым.  

Прожекторы в комнате (даже накладные расходы), которые находятся в представлении, если отслеживание камер иногда может быть неизвестны, что приводит к путанице в контексте AEC/АГК для камер отслеживания. Неструктурированное и рассеянное освещение.
