# <a name="3d-app-launcher-design-guidance"></a>Руководство по проектированию средств запуска трехмерных приложений

Когда вы Помещаетесь на гарнитуру Windows Mixed Reality (VR), вы переходите на домашнюю страницу Windows Mixed Reality. Домашняя страница отображается в виде дома на клиффе, окружающем горы и воду, но вы можете [выбрать другие среды и даже создать собственные](../design/add-custom-home-environments.md). В сфере дома пользователь может размещать и упорядочивать трехмерные объекты и приложения, которые им нужны. **Средство запуска 3D-приложений** — это «физический» объект в мире смешанной реальности пользователя, который можно выбрать для запуска приложения.

![Пример: средство запуска многомерных приложений с плавающей запятой](images/20171016-151526-mixedreality1-1200px-1000px.jpg)<br>
*Пример средства запуска приложения 3D с плавающей высоты (вымышленное приложение)*

## <a name="design-concepts"></a>Принципы проектирования

### <a name="fantastic-yet-familiar"></a>Самое хорошо знакомое

Среда Windows Mixed Reality, в которой находится средство запуска приложений, является знакомой, а часть — более понятная или Sci-Fi. Лучшие средства запуска следуют правилам этого мира. Подумайте о том, как можно взять привычный репрезентативный объект из своего приложения, но с изгибим некоторые правила фактической реальности. Результат будет волшебным.

### <a name="intuitive"></a>Точки

При просмотре средства запуска приложения его назначение — запуск приложения — должно быть очевидным и не должно вызывать путаницы. Например, убедитесь, что средство запуска является очевидным характером приложения, которое не будет путать с дéкором в Клифф дома. Средство запуска приложений должно пригласить людей для касания или выбора.

![Пример: Новая заметка. средство запуска приложений 3D](images/20171016-152145-mixedreality1-1200px-1000px.jpg)<br>
*Последнее Примечание. Пример средства запуска приложения 3D (вымышленное приложение)*

### <a name="home-scale"></a>Масштаб домашней страницы

программы запуска 3D-приложений в Клифф дома и их размер по умолчанию должны иметь смысл с другими «физическими» объектами в пространстве. Если вы размещаете средство запуска рядом, скажем, дома или в какой-либо мебели, оно должно иметь вид дома и на уровне. Хорошей отправной точкой является просмотр того, как он выглядит на 30 кубических сантиметрах, но помните, что пользователи могут масштабировать его в любое время.

### <a name="own-able"></a>Собственное — возможно

Средство запуска приложений должно иметь такое же значение, как объект. Они будут практически соседними с этими объектами, поэтому средство запуска должно быть похоже на то, что пользователь думал, чтобы находить и хранить поблизости.

![Пример: астро деформация 3D App Launcher](images/20171016-132936-mixedreality-1200px-1000px.jpg)<br>
*Пример средства запуска Астро деформации 3D-приложения (вымышленное приложение)*

### <a name="recognizable"></a>Распознаваемое

Программа запуска 3D-приложений должна мгновенно выразить "марку вашего приложения" пользователям, которые ее видят. Если в приложении есть символ звездочки или особенно идентифицируемый объект, мы рекомендуем использовать его в качестве значительной части проекта. В мире смешанной реальности объект будет вырисовывать больше пользователей, чем только эмблема. Распознаваемые объекты быстро и четко обмениваются торговыми марками.

### <a name="volumetric"></a>Объемные

Ваше приложение заслуживает больше, чем просто поместите эмблему на плоскую плоскость и вызовите ее в день. Средство запуска должно иметь вид привлекательного, трехмерного, физического объекта в пространстве пользователя. Хорошим подходом является Представьте, что ваше приложение помаци в день благодарения параде. Спросите себя, что же в том, что люди на самом деле не поступили на улице? ## <a name="tips-for-good-3d-models"></a>Советы по использованию эффективных трехмерных моделей

* При планировании измерений для средства запуска приложения пропускная поддержка для приблизительного Куба с 30 cm. Таким образом, соотношение размера 1:1:1.
* Модели должны иметь 10 000 многоугольников. [Дополнительные сведения о счетчиках и уровнях сведений о треугольниках (Лодс)](creating-3d-models-for-use-in-the-windows-mixed-reality-home.md#triangle-counts-and-levels-of-detail-lods)
* Протестируйте иммерсивное головной телефон.
* Сведения о сборке в своей геометрической модели, где это возможно — не полагайтесь на текстуры для получения подробных сведений.
* Создание закрытой геометрии с жесткой наводами. Нет дыр, которые не моделируются в.
* Использование природных материалов в объекте. Представьте себе создание ИТ в реальном мире.
* Убедитесь, что модель хорошо читается на разных расстояниях и размерах.
* Когда модель будет готова к работе, ознакомьтесь с [рекомендациями по экспорту ресурсов](creating-3d-models-for-use-in-the-windows-mixed-reality-home.md#asset-requirements-overview).

![Модель с незначительными деталями в текстуре](images/20171013-143334-mixedreality-640px.jpg)<br>
*Модель с незначительными деталями в текстуре*

### <a name="what-to-avoid"></a>Чего следует избегать

* Не используйте подробные сведения о высокой контрастности или небольшие, занятые шаблоны и текстуры.
* Не используйте тонкую геометрию — она не работает на расстоянии, и псевдоним будет неплохой.
* Не разрешите, чтобы части модели не превышают размер 1:1:1. При этом создаются проблемы масштабирования.

![Старайтесь не использовать высококонтрастные шаблоны с малым уровнем доступности](images/20171013-143603-mixedreality-640px.jpg)<br>
*Избегайте высокой контрастности, мелких, занятых шаблонов*

## <a name="how-to-handle-type"></a>Как обрабатывает тип

* Рекомендуется, чтобы ваш тип занимать около 1/3 приложения (или более). Тип — это основная вещь, которая дает пользователям представление о том, что средство запуска, на самом деле, является средством запуска, чтобы было замечательно, если оно существенно.
* Старайтесь не делать этот тип очень широким — старайтесь не использовать его в пределах основных измерений запуска приложения (более или меньше).
* Неструктурированный тип может работать, но может быть трудно просматривать из определенных углов и в определенных средах. Вы можете попытаться поместить его в сплошной объект или заднем плане, чтобы помочь в этом.
* Добавление измерения к типу отлично выглядит в трехмерном виде. Заливка сторон типа, более темного цвета, может помочь в удобочитаемости.

## <a name="lighting"></a>Освещение

Освещение для программы запуска приложений поступает из среды Клифф House. Обязательно протестируйте средство запуска в нескольких местах дома, чтобы оно хорошо выглядело как светлое, так и темнее. Хорошая новость состоит в том, что, если вы уже использовали другие рекомендации по проектированию в этом документе, средство запуска должно быть хорошо знакомым для большинства освещения в Клифф дома.

Для проверки того, как средство запуска выполняет поиск различных источников света в среде, это среда, комната мультимедиа, где-то вне и обратно патио (конкретная область с зеленая трава). Еще один хороший тест — поместив его на половину светлой и половинной тени и посмотрим, как он выглядит.

![Убедитесь, что средство запуска хорошо смотрится как в светлых, так и в тенях.](images/20171013-145523-mixedreality-1200px-1000px.jpg)<br>
*Убедитесь, что средство запуска хорошо смотрится как на светлой, так и в тенях.*

## <a name="texturing"></a>Текстуризации

### <a name="authoring-your-textures"></a>Создание текстур

В качестве конечного формата средства запуска 3D-приложений будет использоваться файл. GLBA, который выполняется с помощью конвейера PBR (с физическим рендерингом). Это может быть непростой процесс. Теперь вы можете использовать технический исполнитель, если вы еще этого не сделали. Если вы дивный DIY-ER, то сможете [исследовать и изучать терминологию PBR](https://wiki.polycount.com/wiki/PBR) , а также узнать о том, что происходит, прежде чем приступать к работе, поможет избежать распространенных ошибок.

### <a name="recommended-authoring-tool"></a>Рекомендуемое средство разработки

Мы рекомендуем использовать [веществ по образцу](https://www.allegorithmic.com/products/substance-painter) аллегорисмик для создания окончательного файла. Если вы не знакомы с созданием шейдеров PBR в веществ по образцу, ознакомьтесь с [руководством](https://docs.allegorithmic.com/documentation/display/SPDOC/Tutorials).

(Как вариант [трехмерной](https://3dcoat.com/home/), так и [куиксел Suite 2](https://quixel.se/suite2/)или [Мармосет тулбаг](https://www.marmoset.co/toolbag/) также будут работать, если вы более знакомы с одним из них.)

### <a name="best-practices"></a>Советы и рекомендации

* Если объект запуска приложения был создан для PBR, он должен просто преобразовать его для среды Клифф House.
* Наш шейдер ожидает рабочий поток "металл/грубая" — недействительный шейдер PBR является закрытым факсом.
* При экспорте текстур учитывайте [Рекомендуемые размеры текстур](creating-3d-models-for-use-in-the-windows-mixed-reality-home.md#material-guidelines) .
* Не забудьте собрать объекты для освещения в режиме реального времени — это означает:
  * Избегайте помогут теней — или нарисованных теней
  * Предотвращение помогут освещения текстур
  * Используйте один из пакетов для создания данных типа PBR, чтобы получить правильные карты, созданные для нашего шейдера.
