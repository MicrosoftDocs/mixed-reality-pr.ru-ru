# <a name="app-quality-criteria-overview"></a>Обзор критериев качества приложения

В этом документе описываются лучшие факторы, влияющие на качество приложений смешанной реальности. Для каждого фактора предоставляются следующие сведения.
* Обзор — краткое описание фактора качества и его важность.
* Воздействие на устройства. тип устройства Windows Mixed Reality, на который влияет действие.
* Критерии качества — оценка коэффициента качества.
* Способ измерения — методы для измерения (или взаимодействия) проблемы.
* Рекомендации — общие сведения о подходах, обеспечивающих лучшую работу пользователей.
* Ресурсы — соответствующие ресурсы для разработчиков и проектирования, которые полезны для создания лучших возможностей приложений.

## <a name="frame-rate"></a>Частота кадров

Частота кадров — первый основополагающий уровень стабильности и комфорт пользователя. Частота кадров ниже рекомендуемых целевых объектов может привести к неблагоприятному снижению голограмм, негативно сказывается на белиевабилити опыта и потенциально выносливости глаз. в зависимости от того, какие Windows Mixed Reality совместимые компьютеры поддерживаются, частота целевого кадра для вашего опыта в Windows Mixed Reality иммерсивное гарнитура составляет 60 гц или 90 гц. для HoloLens скорость целевого кадра составляет 60 гц.

### <a name="how-to-measure"></a>Как измерять

* диаграмма частоты кадров в реальном времени предоставляется на [портале устройств Windows](using-the-windows-device-portal.md#system-performance) в разделе "производительность системы".
* Для отладки разработки добавьте в приложение счетчик диагностики частоты кадров. Пример счетчика см. в разделе ресурсы.
* При этом на устройстве могут возникать частоты кадров, пока приложение работает, перемещая головную часть в сторону. Если в голограмме отображается непредвиденное движение с нарушением колебаний, то, скорее всего, причиной является низкая частота кадров или уровень стабильности.

### <a name="recommendations"></a>Рекомендации

* Добавьте счетчик частоты кадров в начале работы по разработке.
* Изменения, которые требуют сброса в частоте кадров, должны оцениваться и надлежащим образом разрешаться как ошибки производительности.

## <a name="hologram-stability"></a>Стабильность голограммы

Стабильные голограммы повышают удобство использования и белиевабилити приложения и создают более удобное для пользователя удобство просмотра. Качество стабильной работы является результатом хорошей разработки приложений и способности устройства понять (отслеживанию) его среды. Хотя Частота кадров — первый основополагающий уровень стабильности, другие факторы могут повлиять на стабильность, включая:

* Использование плоскости стабилизации
* Расстояние до пространственных привязок
* Отслеживание

### <a name="how-to-measure"></a>Как измерять

Людьми устройство и просматривая его работу:

* Переместите заголовок с боковой стороны на сторону. Если на голограммах отображается непредвиденное перемещение, то вероятная причина может быть вызвана низкой частотой кадров или неправильным выравниванием плоскости стабильности.
* Перемещайтесь по голограммам и окружениям, ищите такие варианты поведения, как дорожки и «переход». Этот тип движения, скорее всего, вызван тем, что устройство не отслеживает среду или расстояние до пространственной привязки.
* Если в кадре находятся большие или несколько голограмм, обратите внимание на поведение голограммы с различной глубиной при перемещении головной позиции с стороны в сторону, если шакинесс, вероятно, это вызвано плоскостью стабилизации.

### <a name="recommendations"></a>Рекомендации

* Добавьте счетчик частоты кадров в начале работы по разработке.
* Используйте плоскость стабилизации.
* Всегда Визуализируйте закрепленные голограммы в пределах 3 метров их привязки.
* Убедитесь, что ваша среда настроена для правильного отслеживания.
* Создайте свой опыт, чтобы избежать голограмм с различными уровнями фокусной глубины внутри рамки.

## <a name="holograms-position-on-real-surfaces"></a>расположение Голограммы на реальных поверхностях

Неверное выравнивание голограмм с физическими объектами (если предполагается, что они помещаются в связи друг с другом) — это четкое указание на не объединение голограмм и реальных. Точность размещения должна относиться к потребностям сценария. Например, при размещении в общей области можно использовать пространственное соответствие, но более точное размещение потребует использования маркеров и калибровки.

### <a name="how-to-measure"></a>Как измерять

* Голограммы, размещаемые на пространственном карте, не должны отображаться сильно переноситься над или под поверхностью.
* Голограммы, требующие точного размещения, должны иметь некоторую форму системы маркеров и калибровки, которая точна для требования сценария.

### <a name="recommendations"></a>Рекомендации

* Пространственное соответствие удобно использовать для размещения объектов на поверхностях, когда точность не требуется.
* Для наилучшей точности используйте маркеры или афиши, чтобы задать голограммы и контроллер Xbox (или какой-либо механизм выравнивания вручную) для окончательной калибровки.
* Рассмотрите возможность разбиения очень больших голограмм на логические части и согласования каждой части с поверхностью.
* Неправильное задание расстояния интерпупиллари (IPD) также может влиять на выравнивание голограмм. всегда настраивайте HoloLens для пользователя IPD.

## <a name="viewing-zone-of-comfort"></a>Просмотр зоны отдыха

Разработчики приложений контролируют, где соходят глаза пользователей, размещая содержимое и голограммы с различной глубиной. пользователи, людьми HoloLens, всегда будут иметь 2,0 м для поддержания четкого изображения, поскольку HoloLens экраны зафиксированы на оптическом расстоянии примерно 2,0 м от пользователя. Неправильная глубина содержимого может привести к появлению Visual дискомфорт или выносливости.

### <a name="how-to-measure"></a>Как измерять

* Обычно содержимое должно быть равно 2 m, но не выше 1,25 или более 5 м.

* Стационарное содержимое не должно быть ближе к 50 cm.

### <a name="recommendations"></a>Рекомендации

* Создайте содержимое для оптимального расстояния при просмотре 2 МБ.
* Установите расстояние прорисовки отсечения равным 85 cm с исчезновением содержимого, начиная с 1 МБ.
* Для стационарных голограмм, для которых требуется более близкое к просмотру, плоскость обрезки должна быть не меньше 30 см, а эффект затухания должен начинаться не менее чем на 10 сантиметров с плоскости обрезки.

## <a name="focus-on-holographic-frame-fov-boundaries"></a>Фокус на границах в holographic кадрах (фов)

### <a name="how-to-measure"></a>Как измерять

* Взаимодействующие объекты распознаются как "взаимодействующие"; включая кнопки, меню и содержимое для конкретного приложения. Как правило, при нацеливании на взаимодействующие объекты должны быть визуальными и звуковыми подсказками.

### <a name="recommendations"></a>Рекомендации

* Использование визуальных и звуковых отзывов для взаимодействия.
* Визуальная обратная связь должна различаться для каждого входящего состояния (неактивно, назначено, выбрано).
* Взаимодействующие объекты должны масштабироваться и размещаться для обеспечения бесплатной нацеливания на ошибки.
* Сгруппированные взаимодействующие объекты (например, строка меню или список) должны иметь достаточное пространство для нацеливания.
* Кнопки и меню, поддерживающие голосовую команду, должны предоставлять текстовые метки для ключевого слова Command (см. раздел, например, ")".

## <a name="room-scanning"></a>Сканирование комнаты

Приложения, для которых требуются данные пространственного сопоставления, полагаются на устройство для автоматического получения этих данных с течением времени и между сеансами по мере того, как пользователь исследует среду с активным устройством. Полнота и качество этих данных зависит от ряда факторов, в том числе от количества выполненных пользователем проверок, времени, прошедших с момента исследования и того, были ли объекты, такие как мебель и дверцы, перемещены, так как устройство проверило эту область. Многие приложения анализируют данные пространственного сопоставления в начале работы, чтобы определить, следует ли пользователю выполнять дополнительные действия, чтобы улучшить полноту и качество пространственной карты. Если пользователь должен проверить окружение, при сканировании необходимо указать четкие рекомендации.

* Сообщите пользователю, где можно усовершенствовать данные. Во многих случаях лучше сообщить пользователю, что нужно сделать (например, Взгляните на центр мебели), чтобы получить необходимое качество сканирования.

## <a name="directional-indicators"></a>Индикаторы направления

В приложении для смешанной реальности содержимое может находиться за пределами поля View или перекрыто по реальным объектам. Хорошо спроектированное приложение упростит пользователю поиск невидимого содержимого. Направленные индикаторы предупреждают пользователя о важном содержимом и предоставляют рекомендации по содержимому относительно положения пользователя. ## <a name="data-loading"></a>Загрузка данных

Элемент управления "Ход выполнения" служит для уведомления пользователя о том, что выполняется длительная операция.
